OpenPHEV — UI/UX Direction

Bauhaus form-follows-function + Tufte data-ink ratio. Monochrome UI. Color is data. Single scroll.

Option A: Typographic Purity

Maximum Tufte. The number IS the interface. No chrome, no boxes, no cards. Just typography, whitespace, and a single sparkline. Data breathes on the page like a printed data table.

Option B: Structured Cards

Bauhaus geometric structure. Same monochrome palette and semantic color, but data grouped into contained cards. Clearer visual separation between 12V / EV / Diagnostics. Slightly more chrome, but structured.

Option C: Information Bands

Hybrid approach. Horizontal bands — each system gets a row. Dense, scannable, no cards. Like a Tufte data table crossed with a dashboard. Everything visible in one viewport if possible.
